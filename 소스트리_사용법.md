# 소스트리에서 GitHub Pages 배포하기

## 방법 1: 소스트리 GUI 사용 (추천)

### 1단계: 변경사항 스테이징
1. 소스트리를 열고 **새로고침(F5)** 버튼 클릭
2. 왼쪽 패널에서 **"작업 디렉토리"** 또는 **"Unstaged files"** 확인
3. 다음 파일들이 보여야 합니다:
   - `.github/workflows/deploy.yml` (새 파일)
   - `vite.config.js` (수정됨)
   - `src/main.jsx` (수정됨)
   - `public/404.html` (새 파일)
4. 모든 파일을 선택하고 **스테이징 영역으로 이동** (파일을 드래그하거나 우클릭 > "스테이징")

### 2단계: 커밋
1. 하단의 **커밋 메시지** 입력란에 다음 입력:
   ```
   Add GitHub Pages deployment configuration
   ```
2. **커밋** 버튼 클릭

### 3단계: 푸시
1. 상단의 **푸시** 버튼 클릭
2. **원격**: `origin` 선택
3. **브랜치**: `master` 선택
4. **푸시** 버튼 클릭

---

## 방법 2: 소스트리 내장 터미널 사용

### 1단계: 터미널 열기
1. 소스트리 메뉴에서 **보기 > 터미널 표시** 클릭
   - 또는 단축키 사용 (보통 하단에 터미널 패널이 열림)

### 2단계: 명령어 실행
터미널에 다음 명령어를 하나씩 입력:

```bash
git add .
git commit -m "Add GitHub Pages deployment configuration"
git push origin master
```

---

## 방법 3: 외부 터미널 사용

1. Windows PowerShell 또는 명령 프롬프트 열기
2. 프로젝트 폴더로 이동:
   ```bash
   cd C:\lyb\portfolio_dev-main
   ```
3. 명령어 실행:
   ```bash
   git add .
   git commit -m "Add GitHub Pages deployment configuration"
   git push origin master
   ```

---

## 확인 방법

### 소스트리에서:
- 푸시 후 **원격** 탭에서 최신 커밋이 보이는지 확인

### GitHub에서:
1. https://github.com/cozyoub/portfolio_dev 접속
2. **Actions** 탭 클릭
3. "Deploy to GitHub Pages" 워크플로우가 실행되는지 확인
4. 완료되면 **Settings > Pages**에서 배포 URL 확인

---

## 문제 해결

### 푸시가 안 될 때:
- **강제 푸시**가 필요한 경우: 푸시 버튼 옆 옵션에서 "Force push" 선택

### Git LFS 파일이 있을 때:
- Git LFS 파일은 자동으로 업로드됩니다 (시간이 걸릴 수 있음)
